## Local vs Myriad Decision

This local machine does not expose `qsub`; full training should run on Myriad or another GPU machine. Local runs should only be smoke checks and figure/table generation from existing outputs.

Use Myriad for:

- n=5 10k+ episode budgets
- n=6 runs
- multi-seed matrix
- Dueling/Double/PBRS variants

Use local machine for:

- `py_compile` and import checks
- 5-20 episode smoke training/evaluation
- small `head_to_head.py` smoke runs
- figure/table generation from existing real outputs

## Output Standards

Figures should be generated by `scripts/make_publication_figures.py` from real CSV/JSON files only. Outputs should go to:

- figures: `figures/publication/`
- analysis tables: `qamel/outputs/analysis/`

Required figure set:

- success rate vs budget
- mean return vs budget
- mean steps vs budget
- generation attempts vs budget
- swap attempts vs budget
- truncated fraction vs budget
- head-to-head policy comparison when data exists
- ablation comparison when data exists
- combined multi-panel summary figure

Required tables:

- per-budget summary
- evaluation summary inventory
- head-to-head summary
- ablation summary
- aggregate table with binomial standard errors for success rates and standard errors across seeds where multiple seeds exist
